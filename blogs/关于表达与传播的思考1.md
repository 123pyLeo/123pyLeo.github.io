#### 一、听众动机

听众来，是为了其中某一类：
1. 想学 → 看重「重要性 / 可迁移性」
2. 想获得后发优势 → 看重「方法 / 差异」
3. 想要乐趣 → 容忍信息不严谨
4. 想被认同 → 背景 / 态度相似
5. 想看新鲜事 → 稀少 / 高价 / 受限
6. 想早点离开 → 容忍度极低
结论：
他们追求的是满足自己的目的，不是你的“正确”。

#### 二、顺应与引导

听众顺应的三种来源：
1. 权力顺应：你是谁（身份 / 标签）
2. 多数顺应：别人都信
3. 底层顺应：他们知道「这是不可逆的现实」
高级表达者通常叠加使用，不会只用一种。

#### 三、理解框架

什么是「框架」：
它不是观点，更像听众理解信息的坐标系。
你所有内容，都会被强制塞进这个框架里。
两种典型框架：
1. 约束型框架
   - 聚焦内容
   - 避免跑题
   - 强逻辑、强边界
2. 支持型框架
   - 帮听众理解
   - 降低认知负担
   - 便于共鸣与传播
常见通用框架：
1. 时间：过去 / 现在 / 未来
2. 结构：背景 → 问题 → 方案
3. 角色：顾客 / 竞争者 / 公司

#### 四、表达目标

你一定要能回答这些问题之一：
1. 听完你，我希望观众有什么感觉
2. 视频结束，评论区会长什么样
3. 我想成为什么样的人
4. 我在经营什么样的组织 / 团队

#### 五、避免对抗感

- 不要说：你应该努力
- 要说：我需要一个能长期坚持的方法

听众天然抗拒被命令，但会主动帮助「需求者」。

#### 六、提前处理质疑

常见质疑类型：
1. 没问题 / 没结论 → 空话
2. 有结论没理由 → 毒鸡汤
3. A 导致 B：
   - A 不一定导致 B
   - 没有 A 也有 B
   - B 根本不重要
主动拆解，信任才会上升。

#### 七、问题具体化

三个关键点：
1. 不用大词
2. 不谈大道理
3. 不迷恋大问题
把抽象问题落到：
1. 行为
2. 场景
3. 可观察现象

#### 八、制造意外感

为什么「意外」有效？
因为人是预测机器：
1. 听起来像常识 → 直接忽略
2. 预测失败 → 注意力被劫持
意外来源：
1. 强对比（A vs B）
2. 性质对比（看似相同，实则不同）
3. 硬关联（两个看似无关的东西）
意外不等于否定听众。

#### 九、冷知识的作用

冷知识不是炫耀，更像工具：
1. 吸引注意力
2. 佐证观点
3. 间接告知
